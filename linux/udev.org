* SPACE CADET KEYBOARD

For maximum default Emacs' binding comfort, I remap my keyboard to conform to
the [[http://xahlee.info/kbd/space-cadet_keyboard.html][Space-Cadet]] layout of the lisp machines Emacs was originally written on.

#+CAPTION: The Venerable Space Cadet Keyboard
#+NAME:fig:Space_Cadet
     [[../img/space_cadet.jpg]]

This layout not only makes so much more sense for Emacs, but for most other
applications as well. Typically the Ctrl key is the most commonly used
modifier, followed by Alt (Meta), and then the Super (Windows) key.

Therefore it makes the most sense to me, to have the most used modifiers
accessible with one's strongest digits (the thumbs), and the least used
modifiers using the weakest digit - one's pinkies.

It also means that your modifiers are entirely symmetrical, so you can use them
in the same touch-typing fashion one uses the shift key (ie. one hand presses
modifier, the other, the modified key).

[[https://wiki.archlinux.org/index.php/Map_scancodes_to_keycodes][Archlinux Wiki Article 1]]

[[https://wiki.archlinux.org/index.php/Extra_keyboard_keys#Scancodes][Archlinux Wiki Article 2]]

To find the correct keycodes use the evtest utility with the path to the input
device as the parameter, eg. /dev/input/event0. The keycode is the unique
identifier after the string "value".

#+BEGIN_SRC sh :tangle no
  sudo evtest /dev/input/event0
#+END_SRC

For examples of how to derive the first keyboard type matching rule, check
either /lib/udev/hwdb.d/60-keyboard.hwdb or the same path with a /usr prefix.

Also, /usr/include/linux/input.h may be of use... Good luck

You can get also the keyboard identifier with:

#+BEGIN_SRC sh :tangle no
  cat /proc/bus/input/devices | grep -i keyboard -A 9 -B 1
#+END_SRC

Finally one needs to run:

#+BEGIN_SRC sh :tangle no
  sudo udevadm hwdb --update && sudo systemd-hwdb update && sudo udevadm trigger
#+END_SRC

These links may come in handy:

[[https://github.com/systemd/systemd/blob/master/hwdb/60-keyboard.hwdb][systemd reference hwdb]]

[[https://yulistic.gitlab.io/2017/12/linux-keymapping-with-udev-hwdb/][tricks and tips]]

You can check it worked with:

#+BEGIN_SRC sh :tangle no
  udevadm info /dev/input/by-path/*-usb-*-kbd | grep KEYBOARD_KEY
#+END_SRC

or:

#+BEGIN_SRC sh :tangle no
  udevadm info /dev/input/eventX | grep KEYBOARD_KEY
#+END_SRC

** Microsoft Wired USB

#+BEGIN_SRC conf
  # Microsoft Wired USB Keyboard
  evdev:input:b0003v045Ep0752*
   KEYBOARD_KEY_70029=capslock      # esc   --> caps
   KEYBOARD_KEY_70039=esc           # caps  --> esc
   KEYBOARD_KEY_700e2=leftctrl      # alt   --> leftctrl
   KEYBOARD_KEY_700e3=leftalt       # super --> leftalt
   KEYBOARD_KEY_700e0=leftmeta      # ctrl  --> super
   KEYBOARD_KEY_700e6=rightctrl     # altgr --> rightctrl
   KEYBOARD_KEY_700e7=leftalt       # super --> leftalt
   KEYBOARD_KEY_70065=rightmeta     # menu  --> rightmeta
#+END_SRC

** Apple MacBook Pro OG

#+BEGIN_SRC conf
  # Apple MacBook Pro
  evdev:input:b0003v05ACp0253*
   KEYBOARD_KEY_70029=capslock   # esc       --> caps
   KEYBOARD_KEY_70039=esc        # caps      --> esc
   KEYBOARD_KEY_700e3=leftctrl   # leftcmd   --> leftctrl
   KEYBOARD_KEY_700e7=rightctrl  # rightcmd  --> rightctrl
   KEYBOARD_KEY_700e2=leftalt    # leftopt   --> leftalt
   KEYBOARD_KEY_700e6=leftalt    # rightopt  --> leftalt (to avoid altgr shit)
   KEYBOARD_KEY_700e0=leftmeta   # leftctrl  --> super
   KEYBOARD_KEY_70050=rightmeta  # leftarrow --> super
#+END_SRC
